# Stage 2 Summary

In Stage 2, I focused on scaling the project from a small proof of concept into a larger real-data baseline. The purpose of this stage was to keep the same simple pipeline from Stage 1, but make it learn from a larger amount of Kepler data. I expanded the configuration so the code would request more Kepler targets, allow more light curves per target, and generate more windows from each light curve. This made the dataset much larger and moved the project closer to a realistic research setup.

Once I ran the Stage 2 pipeline on a machine with internet access, the model successfully downloaded and used real Kepler light curves. In my real-data run, the project loaded real Kepler data for 105 light curves across 35 requested targets. That was a major milestone because it meant the model was no longer training only on fallback synthetic curves. It was now learning from real telescope observations, which include the kinds of brightness variation and noise patterns that exist in actual Kepler measurements. Even though the background light curves were real, I still kept the Stage 1 labeling strategy for this stage, which means the positive examples were still created through synthetic transit injection. So at this stage, the model was learning transit-like patterns on top of real Kepler data, rather than learning from fully curated real exoplanet labels.

In this Stage 2 run, the larger dataset produced 2708 labeled examples, with 2031 used for training and 677 used for testing. The model achieved an accuracy of 0.8449, a precision of 0.8839, a recall of 0.7988, and an F1 score of 0.8392. These results gave me a stronger and more realistic baseline than Stage 1. I also kept model saving in place, so the trained model from this larger real-data run was written to disk and could be reused later by the app or by future analysis steps. This means that Stage 2 gave me a saved baseline model that had been trained on a significantly larger amount of real Kepler data, while still using a simple and explainable machine learning approach.

The main reason Stage 2 matters is that it established a realistic baseline for the project. I now have a version of the model that is trained on real Kepler light curves, saved to disk, and ready to be compared against future improvements. The most important next step after Stage 2 would be Stage 3, where I would replace the synthetic injected labels with real Kepler candidate labels so the model begins learning from actual known transit events instead of simulated ones.
